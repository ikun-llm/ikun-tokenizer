# 第一章：BPE 分词算法

> 怎么把"鸡你太美"拆成计算机能懂的碎片

![BPE分词图示](images/01_bpe.png)

---

## 一句话版本

BPE（Byte Pair Encoding）是一种**子词分词算法**——从最小的字符开始，不断把出现最多的"邻居对"合并成新词，最终得到一个大小合适的词表。就像练习生从基本功开始，把常用的动作组合成招式。

---

## 为什么需要分词？

```
计算机不认识汉字，也不认识英文单词。
它只认识数字。

所以需要一个"翻译官"：
  "鸡你太美" → [42, 17, 88, 65]  （token ids）

这个翻译官就是分词器（Tokenizer）。
```

---

## 三种分词方式的对比

### 方式 1：按字符拆（Character-level）

```
"鸡你太美baby" → ["鸡", "你", "太", "美", "b", "a", "b", "y"]

优点：词表很小（几百个字符就够了）
缺点：序列太长！一篇文章变成几万个 token
      模型要处理的东西太多，太慢了

相当于：把一首歌拆成一个音一个音地唱
        "鸡~你~太~美~" 唱得支离破碎
```

### 方式 2：按词拆（Word-level）

```
"鸡你太美baby" → ["鸡你太美", "baby"]

优点：序列短，含义完整
缺点：词表会非常大（中文光常用词就好几万）
      遇到没见过的词就傻了（OOV 问题）

相当于：练习生只学了固定的舞蹈套路
        遇到新歌就不会跳了
```

### 方式 3：按子词拆（Subword-level）⭐ BPE

```
"鸡你太美baby" → ["鸡你", "太美", "ba", "by"]

优点：词表大小适中（几千到几万）
      常见词保持完整，罕见词拆成小片
      不存在"完全不认识"的词

相当于：练习生学了基本功 + 常见组合
        遇到新动作也能用基本功拼出来
        这就是 BPE 的精髓！
```

## BPE 算法详解

### 核心思想

```
从最小的单元（字节/字符）开始，
反复合并出现次数最多的相邻对，
直到词表大小达到预设值。

就像学功夫：
  第1天：学单个动作（出拳、踢腿、转身）
  第2天：把最常一起做的动作合并成招式（出拳+踢腿 = 连环腿拳）
  第3天：把常用招式再合并成套路（连环腿拳+转身 = 旋风三连）
  ...
  直到学会了足够多的招式
```

### 完整示例

用语料 **"鸡你太美 鸡你太美 太美了"** 来演示 BPE 训练过程：

```
═══════════════════════════════════════════════
第 0 步：初始化（把每个字当作一个 token）
═══════════════════════════════════════════════

词表：["鸡", "你", "太", "美", "了"]
编码后：
  "鸡你太美" → [鸡] [你] [太] [美]
  "鸡你太美" → [鸡] [你] [太] [美]
  "太美了"   → [太] [美] [了]

═══════════════════════════════════════════════
第 1 步：统计所有相邻对的出现次数
═══════════════════════════════════════════════

  ("鸡", "你") → 2 次
  ("你", "太") → 2 次
  ("太", "美") → 3 次  ← 最高！
  ("美", "了") → 1 次

═══════════════════════════════════════════════
第 2 步：合并最高频的对 ("太", "美") → "太美"
═══════════════════════════════════════════════

词表：["鸡", "你", "太", "美", "了", "太美"]  ← 新增！
编码后：
  "鸡你太美" → [鸡] [你] [太美]
  "鸡你太美" → [鸡] [你] [太美]
  "太美了"   → [太美] [了]

═══════════════════════════════════════════════
第 3 步：再统计，再合并
═══════════════════════════════════════════════

  ("鸡", "你") → 2 次  ← 最高！
  ("你", "太美") → 2 次  ← 并列最高！
  ("太美", "了") → 1 次

合并 ("鸡", "你") → "鸡你"

词表：["鸡", "你", "太", "美", "了", "太美", "鸡你"]
编码后：
  "鸡你太美" → [鸡你] [太美]      ← 只要 2 个 token 了！
  "太美了"   → [太美] [了]

═══════════════════════════════════════════════
第 4 步：继续...
═══════════════════════════════════════════════

  ("鸡你", "太美") → 2 次  ← 最高！
  ("太美", "了") → 1 次

合并 ("鸡你", "太美") → "鸡你太美"

词表：["鸡", "你", "太", "美", "了", "太美", "鸡你", "鸡你太美"]
编码后：
  "鸡你太美" → [鸡你太美]         ← 只要 1 个 token！
  "太美了"   → [太美] [了]

这就是 BPE！不断合并，常见的词越来越完整。
```

## ByteLevel BPE：处理多语言的秘密武器

ikun 分词器用的是 **ByteLevel BPE**——先把所有文字转成字节，再做 BPE。

```
普通 BPE：
  以"字符"为基本单位
  中文："鸡" 是一个字符
  问题：世界上有几万个 Unicode 字符，初始词表太大

ByteLevel BPE：
  以"字节"为基本单位
  把所有文字先转成 UTF-8 字节（0-255）
  "鸡" → [0xE9, 0xB8, 0xA1]（3个字节）
  
  优点：
    初始词表只有 256 个（0-255 的所有字节值）
    能处理任何语言，包括 emoji 🐔
    不存在"不认识的字符"

  缺点：
    中文每个字要 3 个字节，序列变长
    需要更多合并次数来学会中文组合
    所以中文语料要足够多！
```

```
ikun 分词器的处理过程：

输入："鸡你太美baby🐔"

Step 1: 转成字节
  "鸡"  → [0xE9, 0xB8, 0xA1]
  "你"  → [0xE4, 0xBD, 0xA0]
  "太"  → [0xE5, 0xA4, 0xAA]
  "美"  → [0xE7, 0xBE, 0x8E]
  "b"   → [0x62]
  "a"   → [0x61]
  "b"   → [0x62]
  "y"   → [0x79]
  "🐔"  → [0xF0, 0x9F, 0x90, 0x94]

Step 2: BPE 合并（训练好的规则）
  经过一系列合并，常见的字节组合被合成 token
  → ["鸡你", "太美", "baby", "🐔"]

Step 3: 映射到 token id
  → [1024, 2048, 3721, 256]
```

## Pre-tokenizer：分词前的预处理

```
在 BPE 之前，还有一个"预分词"步骤：

输入："鸡你太美 baby 鸡你太美"

Pre-tokenizer（按空格/标点拆分）：
  → ["鸡你太美", " ", "baby", " ", "鸡你太美"]

为什么？
  防止跨词合并！
  如果不预分词，"美 b" 可能会被合并在一起（没有意义）
  
ikun 分词器用的是 ByteLevel Pre-tokenizer：
  把空格替换为特殊字符 Ġ
  "鸡你太美 baby" → ["鸡你太美", "Ġbaby"]
  这样空格信息也保留了
```

## 本章小结

| 概念 | 说明 | 比喻 |
|------|------|------|
| 分词 | 把文字拆成 token | 把歌词拆成可以唱的音节 |
| 字符级 | 每个字/字母一个 token | 一个音一个音唱，太碎 |
| 词级 | 每个词一个 token | 只会固定曲目，没法即兴 |
| BPE | 从字符开始，合并高频对 | 从基本功组合成招式 |
| ByteLevel | 用字节而非字符做 BPE | 统一度量衡，全球通用 |
| Pre-tokenizer | BPE 之前的预处理 | 先把歌词分段，再拆音节 |

```
记住 BPE 的口诀：
  数一数，谁最多？合并它！
  数一数，谁最多？合并它！
  重复重复再重复——
  直到词表装满为止！

  就像练习时长两年半——
  不断练习，不断合并，越练越熟！🏀
```
